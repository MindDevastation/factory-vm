## Update policy

- Any canonical document change must include explicit scope and affected artifact list.
- Related subordinate artifacts should be updated in the same slice where feasible.
- If deferred, follow-up obligation must be explicit and traceable.

## Review/approval expectations

- Canonical doc changes require reviewer acknowledgment.
- Reviewer must confirm no parallel canon is introduced.
- Reviewer must confirm mode/boundary separation remains intact.

## Evidence-backed expectations

- Update rationale must cite source-of-truth artifact(s).
- Verification evidence (commands/checks) must be captured in PR/report.
- Unverified assumptions must be listed explicitly.
